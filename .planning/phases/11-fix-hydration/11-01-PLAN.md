# Plan: Fix Hydration Issues

**Phase:** 11  
**Goal:** Fix React hydration errors caused by server/client HTML mismatch

---

## Problem

React hydration fails because server-rendered HTML doesn't match client due to localStorage/window access during initial render.

---

## Implementation

### Task 1: Fix EpisodeList sort order hydration

**File:** `web/src/components/EpisodeList.tsx`

**Change:**
```typescript
// BEFORE (line 24):
const [sortOrder, setSortOrder] = useState<'newest' | 'oldest'>(getInitialSortOrder);

// AFTER:
const [sortOrder, setSortOrder] = useState<'newest' | 'oldest'>('newest');

useEffect(() => {
  if (typeof window !== 'undefined') {
    const saved = localStorage.getItem('episode_sort_order');
    if (saved === 'newest' || saved === 'oldest') {
      setSortOrder(saved);
    }
  }
}, []);
```

---

### Task 2: Fix EpisodeCard played status hydration

**File:** `web/src/components/EpisodeCard.tsx`

**Change:**
```typescript
// BEFORE (line 42):
const played = playedEpisodes.has(episode.id) || (typeof window !== 'undefined' && localStorage.getItem(`podcast_played_${episode.id}`) === 'true');

// AFTER:
const [played, setPlayed] = useState(false);

useEffect(() => {
  if (typeof window !== 'undefined') {
    const wasPlayed = localStorage.getItem(`podcast_played_${episode.id}`) === 'true';
    setPlayed(wasPlayed);
  }
}, [episode.id]);

const isPlayed = playedEpisodes.has(episode.id) || played;
```
